# HR 快速上手

这是给没有技术背景的 recruiter / HR 用的说明。你不需要理解脚本，不需要先填表，也不需要自己建文件夹。直接告诉 coding agent 你要招什么人、简历在哪里，agent 会负责整理需求、创建文件夹和执行流程。

## 你需要告诉 agent 什么

最自然的方式是直接粘贴 JD，或者用自然语言描述你的招聘需求。

你只需要说明：

1. 要筛什么岗位。可以粘贴完整 JD，也可以说：“我要招 3 年以上 B2B SaaS 销售，最好卖过 HR 系统。”
2. 简历在哪里。可以是本地文件夹、邮箱、飞书表格、ATS、招聘网站导出或云盘。
3. 有没有硬性要求。比如地点、年限、语言、学历、薪资、一票否决项。

agent 会负责：

- 把你的 JD / 自然语言需求整理成 `job_requirements.md`。
- 如果关键信息缺失，只追问会影响筛选结果的问题。
- 自动创建运行文件夹、`resumes/`、`work/`、`results/`。
- 把简历复制或下载到 `resumes/`。
- 先跑 3-5 份小样本试跑，让你确认口径。

## agent 会自动创建的文件夹

你不用手动创建。每次任务会有类似这样的结构：

```text
screening-run/
├── job_requirements.md   # agent 根据你的 JD/自然语言需求整理
├── resumes/              # agent 收集/复制简历到这里
├── work/                 # 中间缓存，agent 管
└── results/              # 最终结果看这里
```

你最终只需要打开 `results/`。如果你想检查口径，可以看 `job_requirements.md`；`work/` 通常不用管。

## 你可以这样对 agent 说

```text
用 $resume-screening-pipeline 帮我筛简历。
这是 JD：<直接粘贴 JD 或自然语言需求>
简历在这个文件夹：<路径/链接/来源>
先跑 5 份小样本试跑给我看筛选口径，等我确认后再跑全量。
```

如果简历在腾讯企业邮箱：

```text
用 $resume-screening-pipeline 处理邮箱里的简历。
我是腾讯企业邮箱，账号是 xxx@company.com。
请先下载最近 30 天标题里包含“简历”的附件到 resumes 文件夹。
下载完后先跑 5 份小样本试跑。
```

不要把邮箱密码直接写在公开文档或社媒截图里。实际运行时再输入客户端专用密码。

## 5 分钟流程

1. 直接粘贴 JD，或用自然语言告诉 agent 你要招什么人。
2. 告诉 agent 简历来源。
3. agent 整理岗位需求，并只追问必要缺口。
4. agent 创建文件夹、收集简历、先跑 3-5 份小样本试跑。
5. 打开 `results/resume_screening_results.xlsx`，抽检 `筛选总表`。
6. 如果不同意某些 AI 判断，直接在 Excel 的 `人工初筛结果` 和 `人工初筛判断依据` 两列里改，保存文件。
7. 让 agent 读取这个 Excel，修正筛选标准并重评。
8. 口径确认后，agent 跑全量。

## 怎么看结果

- `推荐`：和 JD 明确匹配，通常值得联系或约面。
- `备选`：有相关信号，但还有缺口，适合二线复查或快速确认。
- `不推荐`：相关性弱，或明显不符合硬性要求。
- `需复核`：AI 无法安全判断，可能是文件不可读、信息不够、格式异常；这不是拒绝。

最终判断不要只看 `AI 初筛结果`，还要看匹配结论、过往经历概况和需要注意的点。

## 怎么用 Excel 反向修正筛选标准

`筛选总表` 里有两列专门给你手填：

- `人工初筛结果`：当你觉得 AI 判断不对时填写，比如“其实一般”“不该推荐”“被误杀”“需要复核”。如果你没意见，可以留空。
- `人工初筛判断依据`：写清楚为什么，比如“客户群体偏 IT，不应进入第一优先级”。

你改完并保存 Excel 后，对 agent 说：

```text
我已经在 Excel 里填了人工初筛结果和判断依据。
请读取 results/resume_screening_results.xlsx，根据我的反馈修正筛选标准，并重新评分。
```

agent 会读取你本地保存的 Excel，对比 AI 初筛和你的人工判断，把可复用的修正写回 `job_requirements.md`，再重新生成结果表。

## 小样本试跑后怎么反馈

跑完 3-5 份后，可以直接改 Excel，也可以按这个格式告诉 agent：

```text
小样本试跑反馈：
1. 推荐里，哪些人我认可？哪些太宽了？
2. 不推荐里，有没有误杀？
3. 备选是不是太多/太少？
4. 一票否决项有没有执行到位？
5. 地点、年限、语言、行业这些硬条件有没有看准？
6. 接下来全量筛选请更严格 / 更宽松 / 保持当前口径。
```

详细机制见 `references/pilot-feedback.md`。

## 给 hiring manager 怎么汇报

不要只转发分数。建议说清楚：

- 为什么这个人被推荐。
- 覆盖了哪些 must-have。
- 风险或缺失信息是什么。
- 第一轮电话/面试应该问什么。

## 隐私提醒

这个流程可能会把简历内容发送给模型 API。请只使用公司允许的模型服务，不要在公开 demo、社媒截图或 GitHub 仓库里暴露真实候选人的姓名、电话、邮箱、简历和邮件来源。
